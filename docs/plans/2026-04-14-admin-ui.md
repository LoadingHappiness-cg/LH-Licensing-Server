# Internal Admin UI Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a server-rendered internal admin UI to the existing LH Licensing Server monorepo so internal users can manage customers, products, plans, licenses, installations, and audit history through the canonical licensing API.

**Architecture:** Extend the current Next.js app in `apps/web` instead of creating a new frontend project. Keep business rules and persistence in `apps/api` with Prisma-backed services and expose only admin API endpoints that the UI consumes. Use the existing Entra ID login flow in NextAuth, add role/group checks in the web layer, and keep the UI server-rendered with simple Bootstrap-level forms and tables.

**Tech Stack:** Next.js 14 App Router, NextAuth, Fastify, Prisma, TypeScript, PostgreSQL/MySQL-compatible schema changes, server actions, and server-rendered pages.
